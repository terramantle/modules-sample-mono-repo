# three-tier-pki

A full three-tier certificate authority chain, built entirely with the `tls`
provider. Cloud-free, applies instantly:

```
Tier 1  Root CA          (self-signed)
Tier 2  Intermediate CA  (signed by the Root)
Tier 3  Issuing CA       (signed by the Intermediate)
Leaf    Server cert      (issued by the Issuing CA)
```

This module is **deliberately interesting for the state scanner**: every CA
private key (root, intermediate, issuing) plus the leaf key is stored in state,
so pushing the consuming stack's state to Terramantle should have `trufflehog`
flag a layered set of secrets - a more realistic target than a single key.

Outputs include each tier's certificate, a `ca_bundle_pem` (issuing ->
intermediate -> root) and a `full_chain_pem` (leaf + bundle). Private-key outputs
are marked `sensitive`.

Published to Terramantle as `three-tier-pki/tls`. Releases are driven by your
commit messages - merge a Conventional-Commit change to `main` and CI versions,
publishes, and tags it for you:

```bash
git commit -m "feat(three-tier-pki): describe the change"   # minor bump
git commit -m "fix(three-tier-pki): describe the fix"       # patch bump
```
