# insecure-rds

> ⚠️ **Intentionally insecure.** This module exists to demonstrate Terramantle's
> scan + policy gating. It publishes successfully but is flagged **not
> consumable** until the org's enforcing policies pass (hardcoded credentials,
> public accessibility, no encryption, etc.).

Published to Terramantle as `insecure-rds/aws`. Releases are driven by your
commit messages - merge a Conventional-Commit change to `main` and CI versions,
publishes, and tags it for you:

```bash
git commit -m "feat(insecure-rds): describe the change"   # minor bump
git commit -m "fix(insecure-rds): describe the fix"       # patch bump
```
